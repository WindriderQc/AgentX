                            5 WORKFLOW TEMPLATES
================================================================================

1. BASIC WEBHOOK (3 nodes)
   Webhook → Process → Respond
   Use for: Simple APIs, status checks, quick transformations

2. SCHEDULED JOB (4 nodes)
   Schedule → Fetch → Process → Log
   Use for: Health checks, periodic syncs, scheduled reports

3. CONDITIONAL FLOW (7 nodes)
   Webhook → Validate → IF → Success/Error → Merge → Respond
   Use for: Validation workflows, approval processes, quality checks

4. MULTI-STEP API (9 nodes)
   Webhook → Prepare → API1+API2 (parallel) → Process → Merge → Respond
   Use for: Data enrichment, multi-service orchestration, aggregation

5. ERROR HANDLING (12 nodes)
   Webhook → Context → Risky Op → Check → IF → Success/Error → Log → Respond
   Use for: Production APIs, critical workflows, robust error handling

                             QUICK START GUIDE
================================================================================

For AI Architect:
1. Read: workflow-templates.json
2. Select template based on requirements
3. Customize: workflow_id, credentials, URLs, logic
4. Deploy: ./scripts/deploy-n8n-workflows.sh

For Developers:
1. Browse: jq '.templates | keys' workflow-templates.json
2. Extract: jq '.templates["1_basic_webhook"].workflow' > my-workflow.json
3. Customize all "CUSTOMIZE" markers
4. Validate: jq empty my-workflow.json
5. Deploy: ./scripts/deploy-n8n-workflows.sh my-workflow.json
